Planteamiento del problema.

La institución educativa distrital JOSE ANTONIO GALAN (pueden elegir otro colegio) requiere de un sistema de información que permita el llamado a lista de cada uno de los estudiantes en cada uno de los diferentes cursos.

el sistema debe ser adaptable a dispositivos móviles(obligatorio). Bootstrap, semantic UI , material design Materialize CSS. 



Para ello usted deberá generar una solución informática al problema.

Objetivos específicos

1.       El sistema contará con 2 roles (profesores y estudiantes).

2.       El sistema debe permitir al profesor la creación de un nuevo grupo o curso.

3.       El sistema debe permitir al profesor registrar un estudiante a un grupo o curso.

4.       El sistema debe validar que el estudiante a la hora de ser registrado quede asociado a un grupo o curso.

5.       El sistema debe permitir el llamado a lista de cualquier curso o grupo.

6.       El sistema debe manejar 4 posibles estados al llamar lista (asistió (A), falla(F), retardo (R), falla justificada (FJ) )

7.       El sistema debe permitir la actualización de cualquier dato.

8.       El sistema debe ser lo más completo posible.

9.       La colorimetría debe ser acorde a un complejo educativo, con sus logos etc…

10.   El estudiante podrá consultar las fallas, retardos, fallas justificadas y asistencias en la plataforma.

11.   El sistema debe contar con una interfaz index en donde se evidencia

·       Nombre del colegio.

·       Misión, visión

·       Ubicación maps

·       Formulario de contacto (solo visual no funcional).

·       Información sobre matriculas

·       Estudiantes destacados

·       Foto de profesores

·       Foto de estudiantes

·       Foto de monitores

·       Foto de coordinadores

·       Foto de las instalaciones del colegio

·       Un pequeño bloc con 6 publicaciones académicas

La información debe ser ordenada de la mejor forma. No se recibirá trabajos hechos a medias.



Entregables

1.       Se debe generar los respectivos mockups del sistema en adobe XD o Balsamiq.(antes de proceder con la codificación).

2.       Se debe generar el modelo Relacional de la base de datos que dé solución a la problemática planteada con sus respectivas entidades y atributos (Start UML).

3.       Se debe crear un repositorio en GitHub con dos ramas en la primera rama guardaran Los Mockups y el modelo de la base de datos y en la segunda rama el código del sistema de información.

4.       Se debe generar una presentación del proyecto prueba en donde trataran de vender la solución a sus compañeros y al instructor. (el proyecto ganador recibirá un incentivo).

Aclaraciones.

el sistema no requiere sistema de autenticación(LOGIN).

El proyecto debe ser terminado no se recibirá un proyecto a medias y mal analizado debe generar lo necesario para funcionar de una manera muy cómoda para el profesor y para el estudiante.



la actividad se desarrollará en grupos de proyecto FINAL.

la persona que no vea trabajando, perderá esta nota y tendrá que desarrollar este mismo sistema solo (a) además de una actividad adicional para recuperar.



La actividad debe ser entregada y expuesta para el dia sabado 5 de octubre del 2019.
